# 01 - Planejamento do Trabalho

## Objetivo

Definir o escopo minimo que precisa estar pronto ate **5 de junho de 2026**.

## Mini Atividades

- [ ] Confirmar a dupla responsavel pelo projeto.
- [ ] Definir o link final do repositorio no GitHub.
- [ ] Separar responsabilidades entre implementacao, testes e README-relatorio.
- [ ] Confirmar que a entrega no AVA sera o PDF gerado a partir do README da raiz.
- [ ] Definir o nome do arquivo de disco simulado, por exemplo `filesystem.dat`.

## Escopo Minimo

O projeto deve permitir:

- criar diretorios;
- apagar diretorios;
- renomear diretorios;
- copiar arquivos simulados;
- apagar arquivos simulados;
- renomear arquivos simulados;
- listar o conteudo de um diretorio;
- registrar operacoes em journal;
- manter todos os dados dentro de um arquivo `.dat`.

## Fora do Escopo

- Criar arquivos reais no Windows para representar arquivos do simulador.
- Depender do gerenciador de arquivos do Windows para validar o conteudo interno.
- Implementar permissoes avancadas, usuarios, grupos ou compactacao.

## Criterios de Aceite

- O time consegue explicar o que sera implementado.
- O prazo de 03/06/2026 a 05/06/2026 esta dividido em tarefas pequenas.
- O requisito do `.dat` esta registrado como regra obrigatoria.
